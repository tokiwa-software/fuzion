== About this Document
This document explains Fuzion's realtime systems.

== Glossary

[#jitter]
((Jitter)):: The deviation between when an event is expected to occur and when it actually occurs. For example, if a periodic task is expected to be released every 10 ms at times
0 ms, 10 ms, 20 ms, 30 ms, ...
but is actually released at
0 ms, 11 ms, 19 ms, 32 ms, ...
then the release jitter is
0 ms, +1 ms, -1 ms, +2 ms, ...
respectively.

A realtime system with low jitter produces events close to their expected times. A system with high jitter produces events with larger timing deviations.

[#task]
((Task)):: A unit of work that can be scheduled and executed in a realtime system.

[#event]
((Event)):: An occurrence that happens at a specific time in a realtime system.

[#period]
((Period)):: The intended time interval between successive releases of a periodic activity. For example, a task configured to run every 10 ms has a period of 10 ms. If the first release occurs at time 0 ms, subsequent releases are expected at : 10 ms, 20 ms, 30 ms, ...
The period is a scheduling property describing how often an activity should occur.

[#periodic_task]
((Periodic Task)):: A task that is executed at regular intervals, defined by its period.

[#sporadic_task]
((Sporadic Task)):: A task that is executed irregularly, with a minimum inter-arrival time between executions.

[#deadline]
((Deadline)):: The latest time by which a task must complete its execution to be considered successful.

[#response_time]
((Response Time)):: The time taken from the release of a task to its completion.

[#cycle]
((Cycle)):: Observes the spacing between two consecutive events in a realtime system, irrespective of schedule.

[#precision]
((Precision)):: The smallest time difference that can be distinguished by a measurement system. Precision depends on the resolution of the clock and the frequency at which measurements are taken. Higher precision allows smaller timing variations to be detected and distinguished.
For example, a clock with microsecond precision can distinguish events separated by 1 µs, whereas a clock with millisecond precision cannot reliably distinguish events occurring less than 1 ms apart.

[#affinity]
((Affinity)):: Restriction binding a thread to specific CPU cores.

[#fifo_scheduling]
((FIFO Scheduling)):: A scheduling policy in which runnable tasks of the same priority execute in the order they became ready to run.
Under Linux SCHED_FIFO, a running task continues executing until it blocks, yields, or is preempted by a higher-priority task. Among tasks with equal priority, execution follows first-in-first-out ordering.

[#priority_scheduling]
((Priority Scheduling)):: A scheduling approach in which tasks are assigned priorities, and higher-priority tasks are preferred over lower-priority tasks.
When multiple runnable tasks have the same priority, an additional policy determines their ordering. For example, Linux SCHED_FIFO uses first-in-first-out ordering among tasks of equal priority, while still allowing higher-priority tasks to preempt lower-priority ones.

[#observer]
((Observer)):: A component that records, measures, or analyzes the timing of events in a realtime system. Examples include timers, tracing tools, profilers, schedulers, and measurement code used to collect latency or jitter data.
